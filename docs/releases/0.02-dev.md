# Splendor 0.02-dev Release Notes

## Status

0.02-dev is the local multi-agent runtime plus daemon-control release. It builds
on the 0.01 local kernel baseline and adds safe in-process coordination,
per-agent isolation, local delegation, daemon control, TypeScript client/types,
and inspect-only multi-agent replay.

## Implemented primitives

- Canonical typed local messages with message identity, run scope, source/target
  agents, schema versioning, causal parent, response requirement, and timestamp.
- Local message router with inbox/outbox delivery, queued/delivered/consumed,
  rejected, and expired trace events.
- Agent isolation ledger for per-agent permissions, quotas, allowed message
  schemas, allowed recipients, and replay-visible denial evidence.
- Local delegation model with explicit parent/child run metadata, typed task
  request/response messages, scoped delegated permissions, structured child
  failures, parent cancellation denial, and trace-linked causality.
- Local runtime daemon API for run lifecycle, percept append, trace reads,
  state-head lookup, inspect-only replay, gateway-mediated action submission,
  health, and capabilities.
- TypeScript `@splendor/types` and `@splendor/client` packages for schema-aligned
  daemon control without moving runtime semantics out of Rust.
- Multi-agent replay causal graph output for message lifecycle events,
  parent/child run links, and permission-laundering denial evidence without
  re-executing side effects.
- Docker deployment image for the local runtime surface with `splendorctl`,
  `splendor-daemon`, the Python SDK, example fixtures, OCI image labels, and a
  non-root runtime user.

## Compatibility notes

- Rust, Python, and TypeScript package versions currently report `0.1.0`; the
  milestone release label is `Splendor0.02-dev`.
- Public schemas remain development-stage and provisional until the 0.1 stable
  primitive spec.
- Daemon production authentication, remote node/fleet dispatch, and multi-host
  transport are explicitly outside 0.02-dev scope.
- TypeScript packages are control-plane/client surfaces only. They do not execute
  policies, verifiers, gateways, adapters, state commits, trace writes, or replay.
- The Docker image is a deployment/install artifact for the local runtime. It does
  not make the local insecure daemon safe for remote exposure; daemon production
  authentication and remote transport remain outside 0.02-dev scope.

## Excluded future behavior

- Fleet identity, signed work-order ingestion with cryptographic verification,
  remote transport, state handoff, and trace aggregation are planned for 0.03-dev.
- Governance approvals, escalation, circuit breakers, kill switch, and policy TTL
  are planned for 0.04-dev.
- Physical/edge device orchestration is planned for 0.05-dev.
- Stable compatibility guarantees are planned for 0.1-dev.
