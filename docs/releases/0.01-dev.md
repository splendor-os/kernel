# Splendor 0.01-dev Release Notes

## Status

0.01-dev is the local kernel baseline. It is implemented and hardened as the
foundation for 0.02 local multi-agent work.

## Implemented primitives

- Tenant, agent, run, tick, action, state, and trace identity.
- Local percept intake, policy invocation, constraint evaluation, gateway
  mediation, adapter execution, outcome recording, state commit, and trace
  emission.
- Tenant action, adapter, permission, quota, invariant, precondition, and
  postcondition enforcement before side effects.
- Explicit state graph and snapshot persistence.
- Append-only trace store with ordering and hash-chain metadata.
- Replay that reconstructs local runs without side effects by default.
- Python SDK local hooks and CLI local operations.

## User-visible commands

```bash
splendorctl --version
splendorctl run --config <path> [--cycles <n> | --forever]
splendorctl trace export --db <trace-db> --run <run-id>
splendorctl state head --db <trace-db> --run <run-id>
splendorctl replay --db <trace-db> --state-db <state-db> --run <run-id>
```

## Baseline smoke command

```bash
bash scripts/verify-0.01-baseline.sh
```

## Compatibility notes

- Rust and Python package versions currently report `0.1.0`; the milestone
  baseline label is `Splendor0.01-dev`.
- Public schemas remain development-stage and provisional until the 0.1 stable
  primitive spec.
- Any breaking behavior change after this baseline requires a migration note in
  this changelog or a release/migration document.

## Excluded future behavior

- Multi-agent messages and daemon/API control are planned for 0.02-dev.
- Fleet identity, signed work orders, remote transport, state handoff, and trace
  aggregation are planned for 0.03-dev.
- Governance approvals, escalation, circuit breakers, kill switch, and policy TTL
  are planned for 0.04-dev.
- Physical/edge orchestration is planned for 0.05-dev.
- Stable compatibility guarantees are planned for 0.1-dev.
